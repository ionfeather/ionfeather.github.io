实习 | 大模型推理加速技术探索

2026-04-28
阅读时长：8 分钟

1. 简介

大模型推理加速的目标是在保证模型效果的同时提高吞吐量、降低延迟。目前业务上有非常迫切的模型推理加速需求，因此我们探索了推理加速的前沿技术，主要包括投机采样和模型量化，并且落地到具体的业务场景（舆情分析、涉价值观），取得了不错的推理效果。

2. 技术原理与实验

2.1 投机采样解决的业务问题

目前各个业务场景所使用的大模型，其参数量通常在百亿、千亿级，要把这样大的模型加载进 GPU 中，并且能够在高并发的情况下快速进行推理是非常困难的。我们发现在实际业务场景中，请求模型的 prompt 大部分情况下是相同的，模型在进行 attention 计算时，对这些相同的内容进行了重复计算。因此我们设想能否利用小模型推理速度快、GPU 占用少的特点，先用小模型进行推理，从而释放大模型的压力。

投机采样就是这样一种技术：利用小模型多次推理单个字，让大模型进行多字预测，从而提升整体推理效率。每次小模型的单字推理耗时远远小于大模型，因此投机采样能够有效提高推理效率。其优势在于，通过蒸馏学习和投机采样，可以在减小模型规模的同时，保持较高的预测效果和推理速度，从而在实际部署中获得更好的性能优化。

2.2 投机采样介绍

投机采样的过程如下：

  1. 使用小模型自回归采样生成 n 个 token
  2. 把生成的 n 个 token 和前缀给到大模型执行一次 forward
  3. 对比大小模型的结果，如果发现某个 token 小模型生成的不好，重新采样这个 token，重复步骤 1
  4. 如果小模型生成的 token 都满意，那么接受，继续采样，重复步骤 1

接受和修正的逻辑（对于每个 token，小模型的预测概率为 q，大模型的预测概率为 p）：

  • 若 p ≥ q，则接受当前 token，继续下一个 token 的判断
  • 若 p < q，则以 p/q 的概率接受当前 token，以 (1 - p/q) 的概率拒绝。拒绝之后调整 p 的分布为 norm(max(0, p-q))

为了保证高接受率，小模型需要使用大模型的训练集（SFT 数据和 pretrain 数据集）重新训练一次才可以与大模型对齐。因此投机采样的缺点就是要维持一个小模型的训练，训练成本较高。另一方面，投机采样的采样方式为 Token-Level 采样，其小模型的输入是 Token Embedding。Token Embedding 是一个浅层特征，很难通过一个轻量模型拟合出具备原始 LLM 模型的概率分布，限制了小模型的准确率，意味着验证阶段的接受率也难以提高。

2.4 EAGLE 要解决的业务痛点

使用投机采样需要训练小模型，且小模型需与大模型的训练数据分布一致。目前业务上使用的大模型（如 Qwen）训练数据不公开，收集训练数据困难，导致训练小模型成本较高。另一方面，投机采样使用 token-level 采样，信息较少，小模型推理的接受率低。因此需要一种训练成本低、采样接受率高的方法。经过调研，EAGLE 可以解决这些问题。EAGLE 也是一种投机采样方法，使用 Token-Level & Feature-Level 方式进行 token 采样，且训练成本很低。

2.5 EAGLE 介绍

EAGLE 的 Draft 模型是一个同构的轻量 LLM，其中 Embedding 层和 LM Head 层均复用原始 LLM 模型，中间的 One Auto-regression Head（AR Head）由一层 FC 层以及一层 Transformer Layer 组成。AR Head 是唯一需要微调的网络层，训练成本极低。EAGLE 使用 Draft 模型通过自回归采样生成 Draft Tokens，并且为了提升接收率，在运行 AR Head 前会融合上一个 Token 的隐状态与当前 Token 的 Embedding 层，并通过 AR Head 的 FC 层融合。

[图片：EAGLE 架构示意图]

EAGLE v1 的 draft 树是静态树。token 接受率除了与 Token 所在位置相关，还和上文相关（树中的祖宗节点），v1 的静态树没有利用这一信息。

EAGLE v2 在 v1 的基础上，生成 draft 树时每个节点包含路径的概率信息（通过将路径上边的权重相乘得到）。Expand 过程中，当展开节点的概率值小于阈值时，则停止该节点继续展开，否则继续展开。ReRank 阶段保留概率最高的 K 个节点，其余删除。当不同节点概率相等时，保留浅层节点，以保证树的结构。

[图片：EAGLE v1 静态树 / v2 动态树对比]

2.6 EAGLE 实验

  • 大模型：普通模型 qwen2.5-7B
  • 数据：通用英文数据和中文数据
  • 框架：vLLM
  • 速度指标：同前（并发数、延迟、tokens_per_秒、p50/p95/p99）

性能测试结果：EAGLE 在 vLLM 框架上，英文数据低并发速度提升 50%，高并发速度提升 180%~230%。中文数据上，高并发（50, 100）速度提升 30%~50%，低并发（1, 10）速度降低。原因在于 EAGLE 官方模型在英文数据集上训练，draft 模型在英文数据上接受率高，中文数据上接受率低。若在中文数据上重新训练，效果会改善。

2.7 模型量化要解决的业务痛点

投机采样及其系列方法（EAGLE）使用小模型替代部分大模型推理工作，但缺点是小模型同样占用显存与内存，且无法部署在移动设备、物联网等资源受限环境中。模型量化可以在加速的同时降低模型大小、显存和内存占用。其核心是对模型进行压缩，可能降低一定效果，但能实现加速和节省资源。

2.8 模型量化介绍

大模型训练时，主权重为 FP32。推理时 FP16 权重通常能提供与 FP32 相似的精度，仅需一半 GPU 显存。若要进一步减少显存消耗，可使用 INT8 量化。量化后模型大小降低，推理速度变快，效果不掉点或轻微掉点。

INT8 量化将浮点数 f 通过缩放因子 scale 映射到范围 [-128, 127] 内的 8bit 表示 q：

$$ q = \text{Clip}(\text{Round}(f \times \text{scale})) $$

其中 Round 表示四舍五入取整，Clip 将离群值截断到 [-128, 127]。scale 通常按如下方式计算：

$$ \text{amax} = \max(|x_f|) $$

$$ \text{scale} = 127 / \text{amax} $$

反量化过程为：

$$ x_f' = x_q / \text{scale} $$
